# ТРЕБОВАНИЯ К БЛОК-СХЕМАМ:

Формат вывода: только .drawio (XML для diagrams.net). Никакого SVG, PNG, Mermaid.

## Блоки ГОСТ:
- Овал — «Начало» / «Конец»
- Параллелограмм — ввод/вывод данных
- Прямоугольник с палочками по бокам (shape=process) — вызов функции (предопределённый процесс)
- Прямоугольник — вычисление, действие
- Ромб — ветвление (Да/Нет). ТОЛЬКО для настоящего выбора, НЕ для цикла
- Шестиугольник loop_limit (начало) + перевёрнутый flipV=1 (конец) — любой цикл. Никаких ромбов с петлёй назад
- Маленький круг с цифрой — соединитель (перенос схемы если не помещается)

## Язык в блоках:
- Имена функций и переменных — только на русском
- Всё остальное содержимое блоков — на русском
- Математику лучше избегать английских выражений и все перефразировать на русский: (index//6)+1, new_index = index + шаг
- Слова «encode», «decode», «return», «if» — переводить: «Действие encode?», «Возврат результата»
- В ромбах валидации — конкретное условие: «Шаг больше 32?», а не «Ключ валиден?»

## Структура схемы:
- Шифрование и расшифрование на одной схеме. Два ромба подряд: «Действие encode?» → ветка шифрования, «Действие decode?» → ветка расшифрования. Две отдельные схемы для шифрования и расшифрования, все делается рядом на одном листе, но не скрепляются
- Валидация ключа/шага: ромб сразу после ввода, стрелка «Да» (невалиден) возвращается назад к блоку ввода
- Описывать ВСЁ что есть в коде — никакие функции не пропускать
- Внутренности вспомогательных функций (из utils, text_utils и т.п.) НЕ раскрывать — только блок-палочка с именем и кратким описанием на русском
- Побочные функции внутри модуля шифра (не utils) — каждая как отдельная схема в отдельном .drawio файле

## Визуал:
- Все блоки белые: fillColor=#ffffff, strokeColor=#000000
- Шрифт: fontStyle=1, fontSize=12-13
- Стрелки: edgeStyle=orthogonalEdgeStyle
- У ромбов подписи на стрелках: «Да» / «Нет»

## Алгоритм работы:
1. Прочитай весь код который я дам
2. Составь текстовый план схемы (список блоков сверху вниз с указанием типа каждого) и покажи мне — жди моего подтверждения
3. Уточни у меня: нужна ли схема main/интерфейса отдельно, есть ли валидация ввода в коде, какие функции считать вспомогательными утилитами
4. После подтверждения — генерируй .drawio файл
5. Спроси нужны ли отдельные схемы для вспомогательных функций

## НЕ делать:
- Ромбы для циклов (они только для условий)
- Цвета в блоках
- Две отдельные схемы для шифрования и расшифрования, все делается рядом на одном листе, но не скрепляются
- Английские слова в смысловых блоках, НИГДЕ!! все должно быть на русском
- Расплывчатые условия в ромбах («Ключ валиден?»)
- Пропускать функции из кода, делать их внешними через пунктирную стрелочку
